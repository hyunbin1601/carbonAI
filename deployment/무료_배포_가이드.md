## 📊 현재 프로젝트 구조

```
carbon_ai_ver3/
├── agent-chat-ui/          (프론트엔드: Next.js)
│   └── 포트: 3000
```

## 🚀 방법 1: Railway로 배포 (추천)

### 1단계: Railway 가입

1. https://railway.app 접속
2. "Start a New Project" 클릭
3. GitHub 계정으로 로그인
4. $5 무료 credit 자동 부여

---

### 2단계: 백엔드 배포 (react-agent)

#### 2-1. Railway 프로젝트 생성

```bash
# Railway CLI 설치 (선택사항)
npm install -g @railway/cli

# 또는 웹에서 진행
```

#### 2-2. GitHub Repository 생성

```bash
cd C:\ruffy\carbon_ai_ver3\react-agent
git init
git add .
git commit -m "Initial commit"

# GitHub에서 새 repository 생성 후
git remote add origin https://github.com/yourusername/carbon-ai-backend.git
git push -u origin main
```

#### 2-3. Railway에서 배포

1. Railway 대시보드에서 "New Project" 클릭
2. "Deploy from GitHub repo" 선택
3. `carbon-ai-backend` repository 선택
4. Railway가 자동으로 Python 감지 및 배포

#### 2-4. 환경 변수 설정

Railway 프로젝트 → Variables 탭에서 추가:

```env
ANTHROPIC_API_KEY=your_api_key_here
LANGSMITH_API_KEY=your_langsmith_key
NETZ_MCP_URL=https://hooxi.shinssy.com
NETZ_MCP_ENABLED=true
NETZ_ENTERPRISE_ID=1
```

#### 2-5. 백엔드 URL 확인

Railway가 자동으로 URL 생성:
```
https://carbon-ai-backend-production.up.railway.app
```

---

#### 3-4. 프론트엔드 URL 확인

## 🚀 방법 2: Vercel (프론트) + Render (백엔드)

### 1단계: 백엔드 배포 (Render)

#### 1-1. Render 가입

1. https://render.com 접속
2. GitHub 계정으로 로그인
3. 무료 티어 선택

#### 1-2. GitHub Repository 생성 (위와 동일)

#### 1-3. Render에서 Web Service 생성

1. Render 대시보드 → "New +"
2. "Web Service" 선택
3. GitHub repository 연결
4. 설정:
   ```
   Name: carbon-ai-backend
   Environment: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: langgraph up --host 0.0.0.0 --port $PORT
   ```

#### 1-4. 환경 변수 설정

Render → Environment 탭:
```env
ANTHROPIC_API_KEY=your_api_key
LANGSMITH_API_KEY=your_key
NETZ_MCP_URL=https://hooxi.shinssy.com
NETZ_MCP_ENABLED=true
```

#### 1-5. 백엔드 URL 확인

```
https://carbon-ai-backend.onrender.com
```

⚠️ **중요**: 첫 요청 시 30초 대기 (슬립 모드에서 깨어남)

---

### 2단계: 프론트엔드 배포 (Vercel)

#### 2-1. Vercel 가입

1. https://vercel.com 접속
2. GitHub 계정으로 로그인

#### 2-2. GitHub Repository 생성 (위와 동일)

#### 2-3. Vercel에서 프로젝트 임포트

1. Vercel 대시보드 → "Add New..." → "Project"
2. GitHub repository 선택: `carbon-ai-frontend`
3. Framework Preset: "Next.js" (자동 감지)
4. Environment Variables 추가:
   ```
   NEXT_PUBLIC_API_URL=https://carbon-ai-backend.onrender.com
   NEXT_PUBLIC_ASSISTANT_ID=agent
   ```
5. "Deploy" 클릭

#### 2-4. 프론트엔드 URL 확인

```
https://carbon-ai-frontend.vercel.app
```

**이제 이 URL을 공유하면 끝!** 🎉

## ⚙️ 배포 전 준비사항

### 1. requirements.txt 생성 (백엔드)

`react-agent/requirements.txt` 생성:

```txt
langgraph
langchain-anthropic
langchain-community
chromadb
sentence-transformers
rank-bm25
python-dotenv
fastapi
uvicorn
pypdf
python-docx
httpx
diskcache
```

### 2. 빌드 스크립트 추가 (백엔드)

`react-agent/railway.json` 생성:

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "langgraph up --host 0.0.0.0 --port $PORT"
  }
}
```

### 3. CORS 설정 (백엔드)

백엔드가 프론트엔드 도메인에서 오는 요청을 허용하도록 설정 필요.

## 🔧 문제 해결

### 문제 1: 백엔드 연결 실패

**증상**: 프론트엔드가 백엔드 API 호출 실패

**해결**:
1. 백엔드 URL 확인: `NEXT_PUBLIC_API_URL`
2. 백엔드 로그 확인 (Railway/Render 대시보드)
3. CORS 설정 확인

### 문제 2: API Key 오류

**증상**: "API usage limits" 에러

**해결**:
1. 환경 변수에 올바른 API Key 입력
2. Railway/Render/Vercel에서 Environment Variables 재확인
3. 배포 재시작

### 문제 3: Render 슬립 모드

**증상**: 첫 요청이 30초 걸림

**해결**:
- 무료 티어의 정상 동작
- 대안: Railway 사용 또는 유료 플랜($7/월)

### 문제 4: Railway Credit 소진

**증상**: "Usage limit exceeded"

**해결**:
1. 새 계정 생성 (이메일 다름)
2. Render로 이전
3. 유료 플랜 ($5/월)
